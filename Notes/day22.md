# Day 22 — Systematic Assessment of API Reconnaissance Methodologies

## 1. Foundational Difference: Backend Logic vs. UI
- *Actions:* Distinguished traditional web reconnaissance (UI-driven) from API reconnaissance (logic-driven), focusing on endpoint architecture and object references.
- *Outcome:* Established complete decoupling of reconnaissance phase from exploitation phase as core principle.

---

## 2. Strategic Recon Axis
- *Actions:* Analyzed five critical axes: Identity (JWT/OAuth), Privilege (role hierarchy), Indexing (object identifier predictability), Legacy (deprecated endpoints), Environment (non-production assets).
- *Outcome:* Created comprehensive intelligence framework covering all architectural reconnaissance dimensions.

---

## 3. Hierarchical Recon Layers
- *Actions:*
  - Discovery: Mapped visible surface endpoints.
  - Categorization: Separated functional logic from resource access.
  - Parameter Analysis: Identified control flags and identifiers.
  - Role Mapping: Defined horizontal/vertical privilege boundaries.
  - Trust Integration: Evaluated token-permission synthesis.
- *Outcome:* Built progressive reconnaissance pipeline from surface discovery to complete trust boundary mapping.

---

## Summary
- *Actions:* Concluded reconnaissance as most critical API engagement phase, providing structured intelligence for targeted BOLA, BFLA, and authorization audits.
